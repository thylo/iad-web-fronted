## Note liminaire : le paradoxe assumé

Cette page a été rédigée avec l'aide d'une IA générative. Le paradoxe est évident : utiliser un outil énergivore pour discuter de sobriété numérique. Pourtant, ce paradoxe illustre précisément la tension dans laquelle nous nous trouvons en tant que créateurs numériques.

**L'IA n'est pas le problème en soi.** C'est son usage irréfléchi, systématique et démesuré qui pose question. Un usage responsable et raisonné de l'outil — pour gagner du temps sur des tâches spécifiques, pour explorer des idées, pour apprendre — peut être défendu.

Ce qui compte : **être conscient de l'impact, faire des choix éclairés, ne pas automatiser sans réflexion.**

## L'impact environnemental du web

### Les chiffres qui dérangent

Le numérique représente aujourd'hui :
- **4% des émissions mondiales de gaz à effet de serre** (plus que l'aviation civile)
- **10% de la consommation électrique mondiale**
- Une croissance de 9% par an de son empreinte carbone

**Sources principales :**
- Fabrication du matériel (70% de l'impact)
- Consommation énergétique des datacenters
- Réseau et transmission des données
- Obsolescence programmée

### Le web lourd

Le poids moyen d'une page web a explosé :
- **2000 :** ~50 Ko
- **2010 :** ~700 Ko
- **2024 :** ~2,5 Mo

**Causes :**
- Images non optimisées
- Vidéos en autoplay
- JavaScript surchargé
- Frameworks massifs pour des tâches simples
- Tracking et publicité
- Fonctionnalités inutiles

### Le coût de l'IA générative

**Entraînement d'un modèle :**
- GPT-3 : ~1 287 MWh (équivalent de 120 foyers pendant un an)
- Émissions : ~552 tonnes de CO2

**Utilisation quotidienne :**
- Une requête ChatGPT : ~0,003 kWh (10x plus qu'une recherche Google)
- Impact cumulé massif à l'échelle mondiale

**Mais contexte important :**
- L'entraînement est fait une fois, l'utilisation est le coût récurrent
- Comparé à d'autres activités numériques (streaming vidéo, gaming), l'usage modéré reste gérable
- Le problème est l'usage systématique et la surutilisation

## Lowtech web : faire mieux avec moins

### Principes

Le **lowtech web** n'est pas un retour à l'âge de pierre du HTML 1.0. C'est une approche qui questionne chaque choix technique :

**Est-ce nécessaire ?**
- Ai-je vraiment besoin de ce framework ?
- Cette image doit-elle être en 4K ?
- Cette fonctionnalité sert-elle réellement l'utilisateur ?

**Est-ce sobre ?**
- Puis-je réduire le poids de mes assets ?
- Le code peut-il être plus simple ?
- Y a-t-il des alternatives moins énergivores ?

**Est-ce durable ?**
- Mon site fonctionnera-t-il dans 5 ans ?
- Est-il accessible sur de vieux appareils ?
- Ai-je créé des dépendances fragiles ?

### Pratiques concrètes

**Optimisation des images :**
- Formats modernes (WebP, AVIF)
- Compression adaptée
- Lazy loading
- Dimensions appropriées au contexte

**Code sobre :**
- HTML sémantique (moins de div, plus de balises significatives)
- CSS vanilla plutôt que frameworks lourds
- JavaScript uniquement quand nécessaire
- Progressive enhancement (le site fonctionne sans JS)

**Hébergement vert :**
- Datacenters alimentés en énergies renouvelables
- Serveurs géographiquement proches des utilisateurs
- Mise en cache intelligente

**Design minimaliste :**
- Interfaces épurées
- Typographie système (pas de fonts externes inutiles)
- Couleurs sobres (les écrans OLED consomment moins avec du noir)
- Animations raisonnables

## IndieWeb : reprendre le contrôle

### La philosophie

L'**IndieWeb** est un mouvement qui prône la décentralisation du web et l'autonomie des créateurs.

**Constats :**
- Le web est dominé par quelques plateformes (Meta, Google, X/Twitter)
- Nos contenus ne nous appartiennent plus
- Les algorithmes décident de ce que nous voyons
- La censure et la disparition de contenus sont courantes
- Les données personnelles sont exploitées commercialement

**Principe fondateur :**
> "Votre contenu est le vôtre. Publiez-le sur votre propre domaine."

### Concepts clés

**POSSE (Publish on your Own Site, Syndicate Elsewhere)**
- Publier d'abord sur son site
- Partager ensuite sur les réseaux sociaux
- Garder le contrôle et la pérennité

**Webmentions**
- Notifications inter-sites décentralisées
- Alternative aux commentaires centralisés
- Conversations distribuées entre sites indépendants

**Microformats**
- Annotations HTML pour structurer l'information
- Permet l'interopérabilité entre sites
- Facilite l'agrégation et la syndication

**IndieAuth**
- Authentification décentralisée
- Votre site = votre identité
- Pas besoin de "Se connecter avec Google/Facebook"

### Outils et ressources

**Générateurs de sites statiques :**
- **Astro** (utilisé pour ce site) : Moderne, rapide, flexible
- **Eleventy** : Minimaliste et puissant
- **Hugo** : Ultra-rapide, adapté aux gros projets
- **Jekyll** : Pionnier, intégré à GitHub Pages

**Hébergement simple et éthique :**
- **Netlify** / **Vercel** : Gratuits pour projets personnels, deployment automatique
- **GitHub Pages** : Hébergement gratuit de sites statiques
- **Neocities** : Hébergement gratuit, esprit oldschool web

**Communauté :**
- **IndieWeb.org** : Wiki, ressources, événements
- **IndieWebCamp** : Rencontres et ateliers
- **Chat IndieWeb** : Discussions en temps réel

### Pourquoi c'est important pour nous

En tant que créateurs d'expériences web créatives, l'IndieWeb nous offre :

**Liberté créative totale**
- Pas de contraintes de plateforme
- Pas d'algorithme qui filtre notre travail
- Design et interactions sur mesure

**Pérennité**
- Nos créations ne disparaissent pas si une plateforme ferme
- Archive personnelle de notre travail
- Portfolio durable

**Apprentissage**
- Maîtrise complète de la chaîne de création
- Compréhension des technologies web
- Autonomie technique

**Éthique**
- Pas d'exploitation des données des visiteurs
- Pas de publicité imposée
- Contrôle sur l'accessibilité et l'inclusivité

## Pratiques responsables pour créateurs web

### En tant qu'étudiant·e

**Pendant l'apprentissage :**

✅ **Utiliser l'IA de manière raisonnée**
- Pour comprendre des erreurs
- Pour générer du code boilerplate
- Pour explorer des approches alternatives
- **Mais jamais sans lire et comprendre le résultat**

✅ **Privilégier la simplicité**
- HTML sémantique avant tout
- CSS vanilla quand possible
- JavaScript uniquement si nécessaire

✅ **Optimiser dès le départ**
- Compresser les images
- Tester sur connexions lentes
- Vérifier l'accessibilité

❌ **Éviter les mauvaises habitudes**
- Copier-coller du code sans comprendre
- Installer 10 libraries pour une simple interaction
- Ignorer les performances
- Designer uniquement pour les derniers MacBook Pro

### Dans vos projets créatifs

**Questions à se poser :**

1. **Nécessité** : Cette fonctionnalité est-elle indispensable à l'expérience ?
2. **Sobriété** : Puis-je obtenir le même résultat avec moins de ressources ?
3. **Accessibilité** : Mon projet fonctionne-t-il sur de vieux appareils ?
4. **Pérennité** : Mon code sera-t-il encore fonctionnel dans 5 ans ?
5. **Éthique** : Est-ce que je respecte les données et le temps de mes utilisateurs ?

**Exemples concrets :**

**Au lieu de :**
- Vidéo d'ambiance en autoplay → Image statique ou lecture au clic
- Font Google de 500 Ko → Font système ou variable font optimisée
- Three.js pour un simple effet → Canvas 2D ou CSS animations
- Tracking analytics intrusif → Plausible ou Simple Analytics (privacy-first)

## Conclusion : vers un web intentionnel

Le web responsable n'est pas une contrainte, c'est **une opportunité créative**.

**Contraintes = Créativité**
- Les meilleurs projets naissent souvent de limitations
- Le style lo-fi et minimaliste peut être une esthétique forte
- La simplicité technique force l'ingéniosité narrative

**Éthique = Durabilité**
- Un projet sobre fonctionne plus longtemps
- Un code simple est plus maintenable
- Une approche indieweb garantit la pérennité

**Impact = Responsabilité**
- Nous créons le web de demain
- Chaque choix technique a des conséquences
- Nous pouvons faire mieux

**Le paradoxe de l'IA assumé**
- Oui, cette page a été écrite avec une IA
- Non, ce n'est pas incompatible avec un usage responsable
- Ce qui compte : la conscience, la mesure, l'intention

---

**Le web peut être beau, créatif, surprenant ET sobre, éthique, durable.**

À vous de choisir comment vous voulez le construire.
