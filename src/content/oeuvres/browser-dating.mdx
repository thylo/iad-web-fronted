## Concept et mécanique

Browser.dating renverse la logique du matching traditionnel. Au lieu de profils :
- Vous choisissez des intérêts visuels
- Vous mentez sur vous-même
- Vous vous présentez comme vous aimeriez être

Avec Browser.dating :
- Vous exportez votre **historique réel** de navigation
- Chaque clic, chaque recherche, chaque site visité = portrait fidèle
- L'IA détecte les patterns cachés et croise les intérêts authentiques

## Architecture technique

1. **Export** : Extension navigateur exporte votre historique (Chrome/Firefox)
2. **Traitement** : Données analysées localement sur GPU dédié
3. **Confidentialité** : Aucun historique n'est partagé directement avec d'autres utilisateurs
4. **Match** : IA trouve les utilisateurs avec patterns similaires
5. **Contact** : Chat direct avec vos matches

## Implications pédagogiques

### Identité numérique

Notre historique de navigation est un **palimpseste involontaire** de nos véritables intérêts :
- Ce que nous cherchons (intentions)
- Comment nous cherchons (préférences)
- Quand nous cherchons (rythmes de vie)
- Combien de temps nous consacrons (priorités)

### Données comme matériau créatif

Dries Depoorter utilise les données brutes comme **médium artistique**. Les données ne sont pas visualisées abstraitement, mais transformées en fonction sociale : créer des connexions authentiques.

### Critique des réseaux sociaux

Browser.dating expose le **fossé entre présentation et réalité** sur les applis de rencontre :
- Tinder = Portfolio idéalisé
- Browser.dating = Preuves comportementales

### Enjeux éthiques

- **Privacy by design** : L'IA traite les données en local
- **Échange équitable** : Vous exposez vos données pour une meilleure connexion
- **Trace acceptée** : L'historique de navigation = consentement explicite

## Pourquoi c'est pertinent pour MM3B/RTMF1M

1. **Expérience interactif où les données = interface** : Votre historique devient le contenu
2. **Mise en lumière des systèmes algorithmiques** : Transparence sur comment l'IA fonctionne
3. **Critique du design des apps sociales** : Démontre l'artifice du self-curation
4. **Combinaison art + technologie + critique sociale** : Même approche que Depoorter (Flemish Scrollers, The Follower)

## Connexions avec le cursus

- **Formes d'interactivité** : Forme-machine (interaction utilisateur ↔ système IA)
- **Rôles du public** : Role-soi-même (participation authentique avec données réelles)
- **Interfaces narratives** : Interface point-and-click (sélection de matches, interaction)
- **Types de choix** : Choix-rationnel (l'historique de navigation = choix antérieurs avec conséquences)
- **Structures narratives** : Structure-elastique (processus linéaire avec explorations optionnelles)

## Débats possibles en classe

- **« C'est invasif »** vs. **« Ça crée une authentique connexion »**
- L'historique de navigation révèle-t-il nos "vrais" désirs ou seulement nos intérêts du moment?
- Peut-on vraiment protéger la privacy avec du "local processing"?
- Est-ce que montrer ses vraies données est plus honnête ou plus flippant?
