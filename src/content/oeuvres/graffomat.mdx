## Vue d'ensemble

Le **Graffomat** est un **robot CNC open-source** créé par Niklas Roy qui peint automatiquement au spray sur les murs urbains. Plutôt qu'un projet solitaire, c'est un **écosystème collaboratif** où les artistes locaux de graffiti, les hackers et les artistes numériques ont ensemble expérimenté les possibilités de l'automatisation créative.

## Fonctionnement technique

### Architecture matérielle
- **Axes X-Y** : Deux perceuses sans fil Makita controllant le positionnement
- **Système de spray** : Buse de peinture modifiée, canettes de spray standard
- **Moteur de contrôle** : Arduino + firmware personnalisé pour synchroniser les mouvements
- **Puissance** : Entièrement autonome, pas de connexion électrique permanente

### Flux de création
1. **Fichiers vecteur** : SVG ou formats similaires
2. **Rastérisation** : Conversion en matrices de points (technique CYMK pour les couleurs)
3. **Trajectoires** : Calcul des chemins pour la buse
4. **Exécution** : Spray automatisé sur la surface

### Spécificités techniques
- **Hello World** : Premier test = illustration vectorielle simple
- **Images complexes** : Utilisation de la technique inkjet = 4 matrices de couleurs (Cyan, Magenta, Yellow, Black)
- **Imprécision intentionnelle** : Les variations et imperfections restent visibles, conservant une esthétique DIY

## Contexte de création

Le Graffomat a été développé dans la **scène locale de street art berlinoise**, spécifiquement pour le "Hall of Fame" de Parkaue, un mur légal où les graffeurs expérimentent.

### Collaborateurs et contributions
- **Pixtxa** (hacker/digital artist) : Reverse-engineering du format de fichier
- **Arturo Castro & Carolina Romano** : Expérimentations 3D avec Blender et animations
- **Kati Hyyppä** : Collaboration sur le "Vektorkollektor" (machine de dessin mobile associée)
- **Workshop en ligne** : Participants distants pouvaient contrôler le spray via clavier

## Dimensions narratives

### Structure : Constellation
Le projet fonctionne comme un **réseau hypertexte sans hiérarchie** :
- Différents fichiers vecteur = différentes branches narratives
- Expérimentations locales = nœuds du réseau
- Collaborations à distance = connexions virtuelles
- Aucun chemin "correct" — exploration libre des possibilités

### Rôle du public : Divinité
Les utilisateurs ont un **contrôle omnipotent** :
- Choisir l'image à peindre (contrôle créatif)
- Définir les paramètres (couleurs, taille, vitesse)
- Décider où placer l'œuvre (géographie urbaine)
- Le spray obéit à la volonté du créateur

### Interface : Galeries
L'interface est une **exploration de fichiers** :
- Sélectionner parmi les images vectorielles disponibles
- Visualiser les résultats potentiels
- Modifier et itérer sur les designs
- Chaque fichier est une galerie de possibilités

### Choix : Exploratoire
Les choix du créateur sont **découvertes sans impact critique** :
- Quel fichier essayer ?
- Comment combiner les images ?
- Que faire ensuite ? (aucune conséquence "mauvaise")

## Dimensions collaboratives

### Local (Berlin)
La scène graffiti locale a **adopté et remixé** le Graffomat:
- Expérimentations directes sur le mur
- Développement de nouveaux designs
- Échanges techniques entre Roy et les graffeurs

### Global (Open-Source)
- **Code et documentation** disponibles sur GitHub (github.com/royrobotiks)
- **Reproducibilité** : d'autres peuvent construire leur propre Graffomat
- **Fork et adaptations** : la communauté peut modifier, améliorer, repenser

### Virtuel (Workshop en ligne)
- Participants distants pouvaient **contrôler le spray** via clavier
- Démocratisation du temps de création
- Communauté internationale collaborant sur un projet unique

## Enjeux pédagogiques clés

### 1. Automatisation vs. Authenticité
Le Graffomat pose une question centrale: **"Un robot peut-il faire du street art ?"**
- Le street art valorise l'authenticité, l'habileté, le risque
- L'automatisation supprime la main de l'artiste
- Et pourtant... l'outil crée quelque chose de nouveau, d'hybride

### 2. Démocratisation des outils
Sans Graffomat, créer du graffiti précis demande **années de pratique** :
- La maîtrise des bombes de peinture
- L'anatomie du mouvement
- La gestion du spray
- Graffomat **rend cela accessible immédiatement**

Mais à quel prix ? Qu'avez-vous perdu ?

### 3. Collaboration homme-machine
Roy ne crée pas seul — il travaille avec:
- Les artistes graffitis (l'expertise urbaine)
- Les hackers (l'ingénierie reverse)
- Les artistes numériques (l'imagerie créative)
- **Aucun d'entre eux ne pourrait faire cela seul**

### 4. Open-Source comme philosophie
Par publier le code et les plans, Roy dit:
> "Je crois que partager les connaissances bénéficie à tous. La créativité prospère sur l'ouverture — remixing, réinterprétation, construction sur les idées existantes."

C'est une philosophie radicale pour un artiste.

### 5. La critique subtile de l'automatisation
Le Graffomat n'est pas une célébration aveugle de l'automatisation — c'est une **critique en action**:
- Montre à la fois la puissance ET les limites
- Révèle ce qui se perd dans l'automatisation
- Interroge: à quoi sert vraiment cette machine ?

## Reconnaissance institutionnelle

Roy a reçu une **bourse du Sénat de Berlin** reconnaissant l'importance culturelle du projet — validation que les institutions voient au-delà du "simple robot".
